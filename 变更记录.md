## 2026-07-15 — 修复 Generate 报 "unsafe or invalid" + 目录整理

### 一句话
接入 AI（needs-04）上线后，Generate 偶发/常发报错 `The AI returned an unsafe or invalid result`。已定位为**模型 token 截断**并修复；改动**由 Claude 直接完成并已提交（2b36773）、已部署到 Cloudflare、线上实测通过**。不是通过需求单让你做的，所以你原本不知道，特此同步。

### 根因
`functions/api/generate.js` 里 `MODEL = openai/gpt-5-nano` 是**推理模型**，推理 token 计入 `max_completion_tokens`。原值 `1000` 太小，推理耗尽预算 → 输出 JSON 被截断/为空 → `JSON.parse` 失败 → `requestFormula` 返回 null → 第 180 行返回 502。

### 已做的三处改动（`functions/api/generate.js`，勿回退）
1. **防截断**：新增 `reasoning: { effort: "low" }`；`max_completion_tokens` `1000 → 8000`。
   - 注意：这是**上限不是消耗量**，按实付计费，调高对正常请求零成本；真正的成本闸门是 `DAILY_SPEND_LIMIT_USD`。**不要改回 1000。**
2. **错误可诊断**：`requestFormula` 由返回 `null` 改为返回 `{ result, reason }`，`reason ∈ {ok, truncated, empty, unparseable, invalid}`；502 分支带 `Diagnostic ID` 并 `console.error({diagnosticId, reason})`；`truncated` 时对用户提示 "ran out of room"。**保留这个结构，别改回裸 null。**
3. **每日限额**：`DAILY_SPEND_LIMIT_USD` `0.1 → 2`（原值一毛钱，内测自撞 429）。README 已同步为 `$2/day`。

### 未改动（红线仍在，别动）
前端 `index.html`、access-token 门禁、size/413 限制、真实数据剥离、JSON Schema、`validateModelResult` 校验规则、`data_collection:deny`/`zdr:true`/`require_parameters:true`。

### 验证情况
- 本地用真实 `generate.js` + mock fetch 跑 5 用例全过（ok/unsure/truncated×2/invalid）。
- 已 push（`2b36773 Commit to main`）→ Cloudflare 部署 Success → 线上实测：VPN 开启下可访问，"BCD的数量求和" 正常返回 `=B2+C2+D2` 并填入目标格。

### ⚠️ 已知待优化（暂不修，功能闭环后再做）
**中文歧义需求，AI 会自己猜方向、不澄清。** 例："BCD的数量求和" 有两解——横向按行 `=B2+C2+D2` vs 纵向整列 `=SUM(B2:D6)`；AI 直接选了横向。此为 R2 澄清能力的产品优化点（调 prompt / 加澄清），非 bug。Cooper 决定：等功能完整后再统一优化。
